## 什么是 Agent Skills？

Agent Skills 是一种轻量级、开放的格式，用于通过专门的知识和工作流扩展 AI 代理能力。

其核心是一个包含 `SKILL.md` 文件的文件夹。该文件包含元数据（至少包括 `name` 和 `description`）以及告诉代理如何执行特定任务的说明。Skills 还可以打包脚本、参考资料、模板和其他资源。

```
my-skill/
├── SKILL.md          # 必需：元数据 + 说明
├── scripts/          # 可选：可执行代码
├── references/       # 可选：文档
├── assets/           # 可选：模板、资源
└── ...               # 任何其他文件或目录
```

## 为什么需要 Agent Skills？

Agent 的能力正变得越来越强，但它们往往缺少可靠完成实际工作所需的上下文。Skills 通过将流程性知识以及公司、团队和用户特定的上下文打包为可移植、可版本控制的文件夹，并按需加载给代理，从而解决了这一问题。这使代理具备：

- **领域专业知识**：捕获专门知识——从法律审查流程到数据分析流水线，再到演示文稿排版——并将其作为可复用的说明和资源。
- **可重复的工作流**：将多步骤任务转化为一致、可审计的流程。
- **跨产品复用**：一次构建 skill，即可在任何兼容 skills 的代理中使用。

## Agent Skills 如何工作？

代理通过**渐进式披露**加载 skills，分为三个阶段：

1. **发现**：在启动时，代理只加载每个可用 skill 的名称和描述，足以判断它何时可能相关。

2. **激活**：当某个任务与 skill 的描述匹配时，代理会将完整的 `SKILL.md` 说明读入上下文。

3. **执行**：代理按照说明执行，必要时还可以运行打包的代码或加载引用文件。

只有在任务需要时才会加载完整说明，因此代理可以在仅占用较少上下文空间的情况下保留许多 skills。

## 开放开发

Agent Skills 格式最初由 [Anthropic](https://www.anthropic.com/) 开发，作为开放标准发布，并已被越来越多的代理产品采用。该标准欢迎更广泛生态系统的贡献。
